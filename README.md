# Pricing Card Refactor Checkpoint

## Objective
The goal of this checkpoint is to fix a broken pricing card web component, correct layout issues, and refactor it into a reusable component for future use.

## What You’ll Do
- Identify and fix HTML and CSS errors in the pricing card.
- Make the layout visually appealing and ensure buttons work properly.
- Refactor the component into a reusable format.
- Test the updated component in a browser.
- Submit before and after code, along with the AI prompt you used.

## Steps Taken
1. Copied the broken snippet into Cursor AI.
2. Fixed layout and styling issues using AI suggestions.
3. Refactored the card into a reusable component.
4. Tested the component with multiple pricing plans to ensure it works correctly.

## AI Prompt Used
"Fix the following pricing card HTML and CSS layout issues and refactor it into a reusable component. Ensure proper styling, responsive layout, and functional buttons."

## Submission
Include:
- Original broken code
- Fixed and refactored code
- AI prompt used
